# PR Integration Action Plan Artefact Template

## Purpose

This file is a generated artefact from the `audit` stage of the PR integration skill.

- It captures repo state, PR analysis, and the exact proposed execution plan.
- It is reviewed and approved before `execute` mode runs.
- A new copy is generated for each integration run.

## Integration PR Title Policy

When opening an integration PR:

1. Title must be outcome-first.
2. Include weighted provenance suffix as `(#<a>, #<b>)` where values are merged PR numbers.
3. Do not prefix titles with process labels such as `Batch`, `Round`, or `Wave`.
4. Preferred example: `Improve file browser flow and fix build gate (#12, #14)`.
5. Use linkable `#<PR>` references to trigger GitHub auto-linking.

## Execution Gates

All must pass before `execute` mode:

1. Preflight checks pass.
2. CI checks pass on latest PR heads.
3. Required review policy passes.
4. Working tree is clean.
5. Plan approved by reviewer.
6. Push approval explicitly granted by user.
7. Audit started on `main`.
8. Split plan approved for PRs marked `split-needed`.
9. Force-push plan approved for all listed operations.
10. Label update plan approved and ready to apply before merge.
11. PR communication plan approved, including final fate messaging.
12. Integration branch name matches `integration/<activity-description>`.
13. Integration PR title policy check passes.
14. Integration PR references use linkable `#<PR>` format.
15. Extreme conflict cases include approved defer/close fate and `changes-requested` label plan.
16. Leftover PRs outside automation include `manual-follow-up` label and next-step note.

## Post-Merge Finalization

After all merges complete:

1. `git switch main`
2. `git pull --ff-only`
3. `pnpm test`
4. `pnpm build`
5. Record final status in run summary.

## Merge Message Format (Required)

```text
PR Title (#PRNo.)

PR Description
```

Preserve literal backticks in the message content.

## Commit Message Format (Required)

Use Conventional Commits with bodies that explain what and why.
When rewriting commit messages with backticks, use file-based input to prevent shell interpolation.

## Rollback Plan

If any gate fails in execution:

1. Stop execution.
2. Revert the last integration commit on integration branch.
3. Re-run `pnpm test` and `pnpm build`.
4. Regenerate this artefact with updated state.
